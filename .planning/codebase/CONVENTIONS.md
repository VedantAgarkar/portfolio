# Conventions

**Analysis Date:** 2026-04-14

## Code Style

**Overall:** Proposed — Airbnb / Prettier / ESLint
**Language Specifics:** JavaScript / JSX
**Formatting:** 2 spaces, no semicolons (Proposed)

## Naming

**PascalCase:** React Components, Classes
**camelCase:** Functions, Variables, Properties
**kebab-case:** Files, Folders, CSS Classes

## Patterns

**UI:** Functional Components with Hooks
**Logic:** Injected services or hooks
**Error Handling:** try/catch at API level + Global error boundary

## Commits

**Format:** Conventional Commits (e.g., `feat: [feature name]`)
**Branching:** Main for releases, branch per phase / feature

## Documentation

**JSDoc:** For complex utility functions
**Markdown:** For project architecture and planning
